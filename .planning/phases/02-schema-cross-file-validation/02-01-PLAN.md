<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → file read | File paths come from validate_viz.sh glob expansion (not user text input) — low trust surface |
| Dashboard XML/JSON → JSON.parse | File content parsed but no eval — parse errors are caught and exit 1 cleanly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Spoofing | validate_dash.js file path arg | accept | Paths from validate_viz.sh glob expansion over committed app dirs; not user-supplied text |
| T-02-01-02 | Tampering | vendor/node_modules/ajv/ | accept | Committed to git; same trust level as acorn/cheerio (Phase 1 pattern) |
| T-02-01-03 | Information Disclosure | FINDING: NDJSON on stderr | accept | Findings contain file paths and violation details — all build-time, no runtime secrets |
| T-02-01-04 | Elevation of Privilege | JSON.parse on CDATA content | mitigate | Wrap JSON.parse in try/catch; on error exit 1 and emit stderr warning (never eval) |
| T-02-01-05 | Denial of Service | Very large dashboard XML | accept | Build-time only; no latency SLA; file size bounded by committed test fixtures |
</threat_model>
