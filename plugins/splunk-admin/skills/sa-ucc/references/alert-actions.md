# Alert Actions Reference

Alert actions are defined in the `alerts` array at the top level of `globalConfig.json`.

## globalConfig Structure

```jsonc
{
  "meta": { ... },
  "pages": { ... },
  "alerts": [
    {
      "name": "my_alert_action",
      "label": "My Alert Action",
      "description": "Does something when an alert fires.",
      "iconFileName": "alerticon.png",   // place in package/appserver/static/
      "activeResponse": {                // optional: Adaptive Response
        "task": ["Create", "Update"],
        "supportsAdhoc": true,
        "subject": ["endpoint"],
        "category": ["Information Conveyance"],
        "technology": [{"version": ["1.0.0"], "product": "MyProduct", "vendor": "MyVendor"}],
        "drilldownUri": "search?q=...",
        "sourcetype": "my:alert"
      },
      "entity": [
        {
          "field": "param1",
          "label": "Parameter 1",
          "type": "text",
          "required": true,
          "help": "Enter the first parameter."
        },
        {
          "field": "severity",
          "label": "Severity",
          "type": "singleSelect",
          "options": {
            "disableSearch": true,
            "autoCompleteFields": [
              {"value": "low",    "label": "Low"},
              {"value": "medium", "label": "Medium"},
              {"value": "high",   "label": "High"}
            ]
          },
          "defaultValue": "medium"
        }
      ]
    }
  ]
}
```

## Alert Action Python Script

UCC generates a skeleton at `output/.../bin/my_alert_action.py`. Copy to `package/bin/` and extend:

```python
import sys
import json

def process_event(helper, *args, **kwargs):
    """
    Called for each event when the alert fires.
    helper provides access to event data, parameters, and Splunk session.
    """
    # Access alert action parameters
    param1 = helper.get_param("param1")
    severity = helper.get_param("severity")

    # Access event fields (if available)
    events = helper.get_events()
    for event in events:
        result = event.get("result", {})
        # Process event...

    # Log
    helper.log_info(f"Processing alert with param1={param1}")

    # Make HTTP calls
    response = helper.send_http_request(
        url="https://api.example.com/events",
        method="POST",
        payload={"severity": severity},
        headers={"Content-Type": "application/json"},
        use_proxy=True   # honours proxy settings from Configuration page
    )

    return 0
```

## Adaptive Response

Set `activeResponse` in the alert definition. Required fields:

| Field | Description |
|-------|-------------|
| `task` | List of task types, e.g. `["Create", "Update"]` |
| `supportsAdhoc` | Whether action can be triggered manually |
| `subject` | Target subject types, e.g. `["endpoint", "user"]` |
| `category` | Splunk AR categories |
| `technology` | Array of `{vendor, product, version}` objects |

Full AR category list: https://splunk.github.io/addonfactory-ucc-generator/alert_actions/adaptive_response/

## Files generated by UCC for alerts

- `default/alert_actions.conf` — alert action stanza
- `default/restmap.conf` — REST endpoint registration
- `bin/<name>.py` — alert action skeleton
- `appserver/static/js/build/custom/<name>.js` — UI widget
