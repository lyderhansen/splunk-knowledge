# splunk-agent-dashboard — Design Spec

**Date:** 2026-05-02
**Version:** 0.1.0
**Status:** Draft

## What This Is

A Claude Code plugin called `splunk-agent-dashboard` with 3 skills that help users create, improve, and deploy Splunk Dashboard Studio dashboards. The agent prioritizes visual impact — every dashboard should look professionally designed from the first draft.

## Who It's For

Anyone who uses Splunk and wants better-looking dashboards without being a design expert. The agent reasons like a designer, handles creative judgment calls, and iterates based on user feedback.

## Plugin Structure

```
dashboard-agent/
├── .claude-plugin/
│   └── plugin.json
└── skills/
    ├── create/
    │   └── SKILL.md
    ├── improve/
    │   └── SKILL.md
    └── deploy/
        └── SKILL.md
```

Workspace output directory: `.splunk-agent/dashboard.json` in the user's working directory.

## Skills

### 1. create

**Trigger:** User wants a new dashboard from scratch.

**Flow:**

1. Ask 3-4 questions one at a time:
   - What is the dashboard about?
   - Who is the audience? (executive, analyst, developer...)
   - Do you have data in Splunk already, or should I use mock data?
   - Any brand or style reference? (dark theme, company colors, "like Datadog"...)
2. If the user mentions a brand or style, use WebSearch to find visual references and color palettes. Otherwise pick a strong default theme (dark canvas, accent colors, clean typography).
3. Generate a complete Dashboard Studio v2 JSON with:
   - Absolute layout with intentional spacing
   - Background rectangles for visual depth and sections
   - A mix of viz types (KPI tiles, charts, maps — not just tables)
   - Real SPL queries if user has data, or `makeresults`-based mock data if not
   - Dark canvas with accent colors as default
4. Present the result and explain design choices briefly. Handle iterations ("more charts", "change colors", "add a filter").
5. Save to `.splunk-agent/dashboard.json`.

**Does not deploy.** Tells the user to run `/deploy` when ready.

### 2. improve

**Trigger:** User has an existing dashboard they want to make better.

**Input — three ways:**
- Paste JSON directly into chat
- Point to a local file path
- Pull from Splunk via MCP by dashboard name/ID

**Flow:**

1. Analyze the dashboard across four dimensions:
   - Visual impact: default grey? uniform colors? no spacing?
   - Layout: cramped? logical reading order? grouped sections?
   - Viz choices: tables where a chart would tell the story better?
   - Data: queries well-structured? any that could be simplified?
2. Present a short critique: what's working, what's holding it back. 3-5 bullet points max.
3. Propose 2-3 specific improvements ranked by visual impact.
4. User picks what to fix (or "do all of it"). Generate improved JSON.
5. Save to `.splunk-agent/dashboard.json`.

**Key rule:** Never change SPL query logic unless the user explicitly asks. Visual improvements only by default.

### 3. deploy

**Trigger:** User wants to push a dashboard to Splunk.

**Flow:**

1. Look for `.splunk-agent/dashboard.json`. If missing, tell user to run `/create` or `/improve` first.
2. Check if Splunk MCP server is available. If not, offer fallback: show the JSON for manual copy/paste into Dashboard Studio.
3. Check if the dashboard ID already exists in Splunk via MCP.
   - Does not exist: proceed to create.
   - Already exists: stop and ask: "This dashboard already exists. (a) overwrite, (b) save with new name, or (c) cancel?"
4. On confirmation, push via MCP. Show success message with dashboard path in Splunk.

**Guardrails:**
- Never auto-overwrite. Always ask first.
- Show a summary before deploying (title, panel count, target app).
- If MCP is unavailable, degrade to "here's your JSON" — never error out.

## Tools Required

| Tool | Used by | Purpose |
|---|---|---|
| WebSearch | create | Find design inspiration, color palettes, brand references |
| Splunk MCP (read) | improve, deploy | Fetch existing dashboards, list dashboard IDs |
| Splunk MCP (write) | deploy | Create or update dashboards |
| Splunk MCP (search) | create, improve | Run SPL queries to verify data exists |
| File Write | create, improve | Save dashboard.json to workspace |

## Design Defaults

Every dashboard generated by this agent starts with these baseline choices (user can override any):

- Dark canvas background (not default Splunk grey)
- Background rectangles to create visual sections
- 8pt grid spacing for alignment
- KPI tiles across the top for at-a-glance numbers
- One accent color with tints/shades, not a rainbow
- No more than 8-10 panels per dashboard (cognitive load limit)

## Skill Communication

Skills do not call each other. The user drives the flow:

```
/create  →  user reviews  →  /deploy
/improve →  user reviews  →  /deploy
```

Output is always `.splunk-agent/dashboard.json`. Each skill reads from and writes to this same file.

## Failure Modes

| Risk | Mitigation |
|---|---|
| Overwriting existing dashboard | deploy checks for ID conflict, asks before proceeding |
| MCP not connected | Graceful fallback to JSON output for manual paste |
| SPL queries return no data | Warn user and suggest fixes, don't silently include empty panels |
| User gives vague request | create skill asks clarifying questions before generating |

## Out of Scope for v0.1.0

- Multi-dashboard projects (one dashboard at a time)
- Saved search management
- User/role permissions
- Dashboard versioning or backup
