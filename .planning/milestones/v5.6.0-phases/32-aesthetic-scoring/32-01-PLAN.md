  <name>Task 1: Create score_design.js with 5 aesthetic dimensions</name>
  <files>plugins/splunk-viz-packs/skills/vp-viz/scripts/score_design.js</files>
  <action>
Create score_design.js in vp-viz/scripts/ following the check_design.js ES5 CJS pattern (per D-03). The script takes two CLI args: visualization_source.js path and theme.js path. It reads both files with fs.readFileSync and scores across 5 dimensions per D-01:

1. Gradient usage (0-20): Count createLinearGradient and createRadialGradient calls in the JS source. 0 calls = 0 points, 1 call = 10 points, 2+ calls = 20 points.

2. Typography hierarchy (0-20): Count distinct font size patterns. Look for fontSize assignments, measureText calls, and Math.floor/Math.round/Math.max used near font-related code. Use a regex to extract numeric font size values from patterns like ctx.font assignments (e.g., digits followed by "px"). Count unique sizes found. 1 unique size = 5 points, 2 unique sizes = 10 points, 3+ unique sizes = 20 points.

3. Spacing ratios (0-20): Check for dynamic spacing versus hardcoded pixel values. Count occurrences of dynamic patterns (h *, w *, height *, width *, canvas.height *, canvas.width * — multiplication with dimension variables). Count hardcoded pixel patterns (bare number assignments to x, y, or position-like vars that are not inside a multiplication). If dynamic count is 0 = 5 points, dynamic count 1-3 = 10 points, dynamic count 4+ = 20 points.

4. Color variety (0-20): Count distinct fillStyle and strokeStyle string assignments in the JS source. Extract unique color values (hex, rgb, rgba, or theme token references like t.series, t.accent, t.text). 1-2 unique = 5 points, 3-4 unique = 10 points, 5+ unique = 15 points, then add +5 bonus if t.accent or t.series appears (capped at 20 total for this dimension).

5. Animation presence (0-20): Check for requestAnimationFrame or setInterval patterns. Neither found = 0 points. One type found = 10 points. Both found, or one type found plus _onMouseMove hover handler = 20 points.

Output format (single line to stdout): "  SCORE: {total}/100 (gradient: {g}, typography: {t}, spacing: {s}, color: {c}, animation: {a})"

If an optional third arg [viz_name] is provided, prefix the output: "  SCORE [{viz_name}]: {total}/100 ..."

The script header comment must document usage: `node score_design.js <visualization_source.js> <theme.js> [viz_name]`

Exit 0 always -- scoring is informational, never causes pipeline failure. If a file arg is missing or not found, print a usage message to stderr and exit 2 (same convention as check_design.js).

Use only var declarations, function keyword, no const/let/arrow/template literals. Require only fs and path from Node.js built-ins.
