Rule 9 (PATTERN-01) — content to add as a new `## Rule 9: Compute multi-row layouts bottom-up, not top-down`:
The test51 working snippet (ES5 only — var/function, no const/let/arrow/backticks):
```javascript
var legendH      = 50;
var legendTopY   = h - legendH - 8;
var captionY     = legendTopY - 16;
var valueY       = captionY - 12;
var gaugeBottomY = valueY - 56;
var gaugeMaxR    = Math.min(w * 0.32, (gaugeBottomY - 60) * 1.0);
```
Symptom callout to state: "elements collide at small panel heights — invisible during dev at one
test height, breaks in production at a different panel size." Anti-pattern to name (top-down):
`gaugeBottomY = h - 76; valueY = gaugeBottomY + 38; ...` — pins the bottom row first then walks
upward by fixed offsets, so when h shrinks the rows overlap.

Rule 5 implementation pattern (PATTERN-02) — add as a sub-section UNDER or immediately AFTER
Rule 5 (after line 134, before "## Rule 6" at line 136). Title it e.g.
`### Rule 5 implementation pattern — shared _render<X> helper`. The test52 pattern (ES5):
```javascript
_renderDark:  function(ctx, data, t, w, h, opt) { t = this._resolveTheme(t, opt); this._renderShared(ctx, data, t, w, h, opt, false); },
_renderLight: function(ctx, data, t, w, h, opt) { t = this._resolveTheme(t, opt); this._renderShared(ctx, data, t, w, h, opt, true);  },
_renderShared: function(ctx, data, t, w, h, opt, isLight) {
  // shared geometry both paths legitimately need
  if (!isLight) { /* dark-only ambient glow / carbon-fiber overlay */ }
}
```
CRITICAL nuance to document explicitly: this does NOT violate Rule 5's "two separate code paths".
Rule 5 forbids a single parameterized render that DERIVES light from dark by dimming effects.
The shared helper is the legitimate common geometry both paths need; theme-dependent EFFECTS are
properly branched with `if (!isLight)`. A reader must not think this pattern contradicts Rule 5 —
state the distinction in one sentence.
