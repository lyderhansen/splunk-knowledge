The entry structure (following the same pattern as Line Chart, Leaderboard, etc.):

**Description paragraph:** "Expresses: correlated multi-signal time-series. F1 telemetry, patient vital signs, network operations." Explain the pattern: stacked horizontal strips sharing a time x-axis, each channel with its own independent y-scale. Per D-01: hover on one channel shows a vertical cursor line across ALL channels at the same x position (synchronized crosshair). This is the F1 lap analysis pattern — throttle, brake, speed, gear, ERS stacked vertically with synchronized time scrubbing.

**Accessibility:** B — multiple dense line charts stacked. Provide channel labels with current values on hover.

**When NOT to use:** Fewer than 3 signals (use multi-series Line Chart instead). More than 6 channels — each strip becomes too thin to read at standard panel sizes per D-02. Unrelated metrics with no shared time axis.

**Technical rules (Canvas rendering guidance per D-03):**
- Divide canvas into equal-height strips: `var stripH = Math.floor((h - timeAxisH - topPad) / channelCount);` where timeAxisH is the shared x-axis area at the bottom
- Each strip gets its own y-scale: compute `yMin` and `yMax` per channel from data, then `var yScale = stripH / (yMax - yMin);`
- Draw thin horizontal divider lines between strips using `t.border` or `t.textDim` at 0.3 alpha
- Left-side label area (~60-80px): draw channel name in `t.textDim` and current value at cursor position in `t.text` — values update as cursor moves
- Y-axis ticks render INSIDE the strip (right-aligned text at 2-3 tick positions), not in a separate axis column per D-02
- Shared time x-axis at the bottom: single row of time labels spanning full canvas width
- Synchronized crosshair per D-01: on mousemove, compute x position from event, draw a vertical line spanning ALL strips from `topPad` to `h - timeAxisH` using `t.accent` at 0.5 alpha. Update the left-side current-value labels for all channels based on the data value at that x position.

**Design rules:** Reference design-principles.md: DPR-01 (typography hierarchy for channel labels), DPR-03 (gradient fills on area-fill channels). Reference consistency-grid.md for spacing and typography scale.

**Creative decisions YOU make:**
- Per-channel line style (solid line, area fill, step line) based on data type and brand personality
- Channel color assignment strategy (brand series colors s1-s6 or accent-derived)
- Crosshair line styling (solid vs dashed, thickness 1-2px)
- Whether strips have subtle alternating background tints for readability
- Time axis label density and format
- Channel label font weight and positioning within the left label area

**Settings:** `channels` (CSV of field names), `channelLabels` (CSV of display names), `channelTypes` (CSV of render types: line/area/step), `xField` (default _time), `showCrosshair`, `showChannelLabels`, `crosshairColor`, `showGrid`, `themeMode`, `accentIntensity`, `showEntrance`, `showHoverEffect`, `animationSpeed`

**Animation notes:** Entrance uses Generic Entrance Boilerplate from animation-recipes.md — globalAlpha fade-in on all strips simultaneously. flashCritical is NOT a default for this viz type (no status values). showHoverEffect controls the synchronized crosshair visibility.

**Data contract:** Time-series with 3-6 numeric value columns. xField (default _time) is the shared time axis. Each additional numeric column is one channel. Multi-row.
**Expected columns:** `| timechart span=1m throttle brake speed gear ers` — _time + 3-6 numeric series
**Drilldown:** crosshair-based — click passes the x-axis value (timestamp) at the click position. Store rendered x-positions array. _onClick finds nearest x-position to mx, passes the _time value.

**Worked example (F1 telemetry, 5 channels):**
Include a brief example showing a 5-channel F1 configuration: Throttle % (0-100, area fill), Brake % (0-100, area fill), Speed km/h (0-350, line), Gear (1-8, step line), ERS % (0-100, area fill).
