### Gaps Summary

**GAP 1 — BLOCKER: B10 false positive for `"drilldown": "all"`**

This is an INT-01 blocker. The phase goal is to add drilldown token flows to generated dashboards. The correct way to enable drilldown on a custom viz is `"options": {"drilldown": "all"}` — this is explicitly documented in the new dashboard-interactivity.md reference. However, validate_dash.js B10 treats any bare key (no dot) in a custom viz's options as a namespace violation and emits FAIL B10. The `"drilldown"` key is a DS v2 system key, not a viz-namespace key, so it must NOT carry the `app.viz.` prefix. Without a whitelist fix, a generated dashboard following INT-01 patterns will fail packaging validation on every run.

The fix requires two lines: a whitelist constant declaration and an indexOf guard before the emitFail call.

**GAP 2 — WARNING: accentColor in 16 viz-blueprints.md Settings: lists**

This is an INT-01/INT-02 friction issue. vp-viz SKILL.md line 158 directs Claude to "Consult viz-blueprints.md Settings: list for the viz type" when generating formatters. All 16 Settings: lists still include `accentColor`. Claude will generate a formatter with accentColor picker, which formatter-patterns.md WRONG patterns table says is removed. This is an internal contradiction in the skill set that will produce confusing but not immediately broken behavior (the accentColor option is silently ignored at runtime).
