The file must contain exactly these 7 sections in order:

**Section 1: Drilldown token flow (setToken baseline) — INT-01**
Document the `drilldown.setToken` eventHandler shape. Key rules verbatim (per D-01, D-03, RESEARCH.md Pattern 1):
- `tokens` is an array of `{token, key}` objects — NOT a map (WRONG pattern)
- `key` is bare (`click.value`, `row.host.value`) — NO `$...$` wrapping
- `options.drilldown: "all"` is required on the panel — omitting it means clicks fire nothing
- Tables use `row.<field>.value`, charts/custom vizs use `click.value`

Embed the verified JSON shape from PATTERNS.md (setToken eventHandlers block with `"options": {"drilldown": "all"}`).

**Section 2: switchToTab — INT-01 (D-02)**
Document the two-handler array pattern: setToken runs first (array order), then `drilldown.switchToTab` with `{ "layoutId": "tab_detail" }`. Note: use for executive-to-analyst drill-in flows. Embed the multi-handler JSON from PATTERNS.md Pattern 2.

**Section 3: resetTokens / unsetTokens — D-04**
Document `drilldown.resetTokens` and `drilldown.unsetTokens` as available handler types. Availability note only (per D-04) — no full pattern required, these are supplemental options.

**Section 4: Input controls — INT-02 (D-05, D-06)**
Subsection A — `input.timerange` (mandatory on every generated dashboard, per D-05): embed the verified JSON from PATTERNS.md Pattern 4. Key rule: `defaultValue` is comma-separated string (`"-24h@h,now"`), NOT an object. Read via `$global_time.earliest$` / `$global_time.latest$` subfields — never bare `$global_time$`.

Subsection B — `input.dropdown` (domain-specific, per D-05): embed Pattern 5 from PATTERNS.md. Include the SPL consumption pattern: `WHERE category="$selected_category$" OR "$selected_category$"="*"`.

Subsection C — Input placement rule (per D-06): global inputs by default; per-tab only if tabs have truly different data domains. Every input key declared in `inputs` MUST appear in `layout.globalInputs` or the relevant `layoutDefinitions[n].inputs` array — declared but unplaced is dead JSON.

**Section 5: defaults block wiring — INT-03 (D-07, D-08)**
Embed the full `defaults` block JSON from PATTERNS.md Pattern 3, showing both `dataSources["ds.search"].options.queryParameters` (time range fan-out) and `tokens.default` (wildcard token initialization). The `"default"` bucket name is literal — document this explicitly.

**Section 6: Domain time defaults — D-08**
A lookup table with three rows:
| Domain | Default value | Notes |
|--------|--------------|-------|
| SOC / NOC | `-24h@h,now` | Real-time threat visibility |
| Executive / trend | `-7d@d,now` | Longer trend window |
| Real-time / streaming | `-15m@h,now` | Near-real-time posture |

**Section 7: SPL consumption patterns**
Document the token consumption WHERE clause pattern:
- Single-value filter: `WHERE field="$tok$" OR "$tok$"="*"` (wildcard default shows all before click)
- Note the `|s` multiselect variant for comma-separated multi-select inputs (for reference; dropdown is single-select by default)

**WRONG patterns section (end of file):**
Copy the WRONG patterns table from PATTERNS.md verbatim (5 entries: map-form tokens, form. prefix, bare timerange token, input not in globalInputs, missing drilldown:all).
