  Content for ECR-08:

  **Requirement:** Any viz that inserts search data into an HTML context MUST escape it first. SPL results are untrusted input -- a malicious search result containing script tags or javascript: URLs will execute if rendered raw. Use `SplunkVisualizationUtils.escapeHtml()` for HTML content and `SplunkVisualizationUtils.makeSafeUrl()` for URL construction.

  **When this applies:** Tooltip HTML (innerHTML assignments), label text that goes through innerHTML or insertAdjacentHTML, dynamic image sources, link href construction, any context where a string from data.rows ends up in a DOM property that interprets HTML/URLs. Note: Canvas 2D ctx.fillText() is inherently safe (renders text, not HTML) -- escapeHtml is NOT needed for pure Canvas text rendering. It IS needed when the viz creates DOM elements for tooltips or overlays.

  **Wrong pattern:** Show raw `row[colIdx['label']]` being assigned to `el.innerHTML` or used in URL construction without escaping.

  **Correct pattern:** Show importing the utilities from SplunkVisualizationUtils:
  - `var escapeHtml = SplunkVisualizationUtils.escapeHtml;` (at module scope, after the require)
  - `var makeSafeUrl = SplunkVisualizationUtils.makeSafeUrl;` (at module scope)
  - Then `tooltip.innerHTML = escapeHtml(safeStr(row[colIdx['label']]));` for HTML contexts
  - And `img.src = makeSafeUrl(safeStr(row[colIdx['icon_url']]));` for URL contexts

  **Rules:**
  - Import both utilities at module scope alongside the SplunkVisualizationBase require
  - escapeHtml every string from data.rows before innerHTML/insertAdjacentHTML
  - makeSafeUrl every string from data.rows before assigning to href/src/action
  - Canvas ctx.fillText() does NOT need escapeHtml -- Canvas text rendering is not an HTML context
  - safeStr() and escapeHtml() serve different purposes: safeStr handles null/undefined, escapeHtml handles HTML entity encoding. Use both: `escapeHtml(safeStr(val))`

  Keep the section concise -- aim for 50-60 lines matching the density of ECR-06 and ECR-07.

  Also update the file's opening paragraph (line 2) to change "ECR-01 through ECR-05" to "ECR-01 through ECR-08" since ECR-06 and ECR-07 already exist and ECR-08 is being added.
