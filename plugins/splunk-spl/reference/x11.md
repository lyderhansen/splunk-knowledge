# x11 — seasonal decomposition of time series (X11 method)

Source: Splunk Search Reference 10.2.0.

## Syntax

    | x11 [<type>] [<period>] (<fieldname>) [AS <newfield>]

The type and period are encoded directly into the function call syntax, e.g. `mult15(count)` means multiplicative with period 15.

## Parameters

| Parameter | Required | Default | Description |
|---|---|---|---|
| `fieldname` | Yes | — | Numeric field in a time-series result set (output of `timechart`) |
| `type` | No | `mult` | Seasonality type: `add` (additive) or `mult` (multiplicative) |
| `period` | No | auto-detected | Integer 5–1000: number of data points in one seasonal cycle |
| `AS newfield` | No | — | Rename the output field; if omitted, output goes to the original field |

## Usage

- Must follow `timechart` or another command that produces a regular time series.
- The period is specified as part of the type/period combination: `mult15(count)` means multiplicative with period 15, `add20(count)` means additive with period 20.
- If period is omitted, Splunk auto-calculates it. The algorithm requires period ≥ 5 and period ≤ 1000.
- Outputs deseasonalized (trend) values, replacing the original field values.

## Examples

### Multiplicative decomposition, period 15 days

    index=download | timechart span=1d count(file) AS count
    | x11 mult15(count)

Because `span=1d`, each data point is one day, so period 15 = 15-day cycle.
`| x11 15(count)` is equivalent since `mult` is the default.

### Additive decomposition, period 20 days

    index=download | timechart span=1d count(file) AS count
    | x11 add20(count)

### Auto-detect period, keep result in new field

    index=sales | timechart span=1w sum(revenue) AS weekly_rev
    | x11 weekly_rev AS trend_rev

## Gotchas

- **Must have ≥ 2× the period in data points** — with too few data points the algorithm fails silently or produces incorrect output; use a wider time range or smaller span.
- **Period ≥ 5 is mandatory** — values below 5 cause errors.
- **Period > 1000 is too slow** — for very long cycles, downsample your data first.
- **Multiplicative x11 requires all-positive values** — if the field ever reaches 0 or negative, use `add` type instead.
- **Input must be a regular time series** — missing time buckets break the algorithm; use `timechart` with `span` to ensure uniformity.
