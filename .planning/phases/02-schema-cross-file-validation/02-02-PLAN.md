<objective>
Extend validate_ast.js with a new --cross <formatter.html> <visualization_source.js> mode that performs cross-file option name consistency checks. It extracts every option name declared in formatter.html (via cheerio DOM parse of name= attributes) and every config key read in visualization_source.js (via acorn AST walk for opt() CallExpression calls), then diffs them. Orphaned formatter options (not read in JS) are FAIL XFILE. JS reads with no formatter declaration are WARN XFILE. Extend test_validate_ast.js with the corresponding --cross test cases.

Purpose: VAL-05 — Cross-file consistency check catches namespace mismatches before Splunk install.
Output: validate_ast.js (modified), test_validate_ast.js (modified).
</objective>
