<objective>
Refactor animation-recipes.md to eliminate the opt()-inside-helper anti-pattern.

Purpose: Claude copies animation boilerplate verbatim. When the boilerplate shows helpers calling opt(config, ns) internally, Claude replicates that pattern and produces viz code where opt() is called outside updateView — causing silent runtime failures because opt() is a closure only available inside updateView scope. A top-of-file rule plus corrected helper signatures removes the only existing counter-example in the codebase, so every future copy produces correct code by default.

Output: animation-recipes.md with (1) a new ## Animation Helper Scope Rule section (D-03), (2) five helper definitions refactored to accept primitives instead of config/ns (D-01 + D-02), (3) LED pulse cadence scaled by speedMult in AB-02 and ANI-02 (D-04), (4) cross-file grep confirming no signature-level changes needed in viz-blueprints.md / canvas-recipes.md / mood-recipes.md (D-05).
</objective>
