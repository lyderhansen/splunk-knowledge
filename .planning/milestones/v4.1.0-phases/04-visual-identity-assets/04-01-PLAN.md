<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg → filesystem | app_dir argument from shell invocation traverses to real filesystem paths |
| theme.js require() | User-supplied theme.js is executed via Node require() — arbitrary code execution surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | generate_assets.js app_dir arg | mitigate | path.resolve() normalizes path before fs.existsSync(); rejects non-existent dirs with exit 1 before any writes |
| T-04-02 | Information Disclosure | require(theme.js) | accept | theme.js is a project-local build artifact written by the developer; arbitrary execution is expected behavior (same risk as build_flat.js) |
| T-04-03 | Tampering | hex color from theme.js | mitigate | isHex() guard validates ^#[0-9A-Fa-f]{6}$ before parseInt; invalid color falls back to neutral gray [128,128,128] |
| T-04-04 | Denial of Service | Large app_dir with many vizs | accept | Local build tool — DoS is a non-issue; CPU/memory usage scales linearly with viz count |
</threat_model>
