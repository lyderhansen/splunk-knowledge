## Decisions Made
- Used `vizId='defaults'` in the DS5 emitFail call because the missing entry belongs to the `defaults` block — consistent with DS2/DS3/DS4 which use `'layout'` for their cross-structure checks
- Chose two-pass approach (collect setTokenNames first, then verify each) over in-place per-handler check for clarity and to avoid duplicate emitFail calls when a token appears in multiple handlers
- Test assertion count is 8 (not 3) — three scenarios with 5/2/1 assertions respectively; plan referred to "3 tests" meaning 3 scenarios
