<objective>
Eliminate the vp-viz side of the Dashboard Studio formatter contradiction. vp-viz currently instructs authors to create a 4th Classic `<form section-label="Effects">` section (SKILL.md item 4, formatter-patterns.md section example + label table + 3 accentColor references, pre-code-checklist "4 sections" line). Dashboard Studio silently drops any non-standard `section-label`, so those controls vanish from the DS config panel — while cv-create's `formatter-emission.md` already (correctly) forbids exactly this. This plan makes vp-viz emit ONLY the three standard Classic labels, folds the Effects toggles + accentColor picker into "Color and style", enforces the rule in the pre-code checklist, states it applies to any Classic viz (incl. hand-authored), and flags (without changing) the separate Extension-API editorConfig question.

Output: four edited vp-viz markdown files + a plugin version bump, with the Classic "Effects" section gone and the accentColor CP-03 usage contract preserved.
</objective>
