Step 1 — Modify generateGradientBg to accept a light theme and produce bg_gradient_light.png:
Change signature from generateGradientBg(appDir, dark) to generateGradientBg(appDir, dark, light). Add a second pixel-grid generation using light theme: light.bg as base (hexToRgb(light.bg)), light.panel (or '#FFFFFF') as the bottom-right color at 0.4 blend (lighter than dark's 0.6), and light.accent radial glow at accentR=400 (smaller, more delicate). Write to bg_gradient_light.png alongside bg_gradient.png.

Step 2 — Add generateSolidBg(appDir, dark, light):
Dark variant: makeRgbRows(1920, 1080, r, g, b) using dark.bg color, then apply subtle noise using a linear congruential generator (seed=12345, multiplier=1664525, increment=1013904223, mask=0xFFFFFFFF). Noise fraction=0.15 (15% of pixels perturbed by delta = ((lcg & 0xF) - 7) in range -7..+8). Write to bg_gradient.png.
Light variant: same approach with light.bg (#F0F2F5 or white). Write to bg_gradient_light.png.

Step 3 — Add generatePatternBg(appDir, dark, light, bgPattern):
Start with makeGradientRows for the base (same as gradient type). Then overlay the pattern on top using fillRect calls:
- 'circuit' (default and most implementable): draw a grid of horizontal and vertical line segments on top. Grid spacing = 80px. Each cell: one horizontal segment (width 40px, height 2px) and one vertical segment (height 40px, width 2px) at the cell corner. Color = dark.accent, blended at 12% opacity via per-channel math: finalR = Math.round(baseR + (accentR - baseR) * 0.12). Since we're working with raw rgbRows, compute per-pixel: for each segment pixel position, read current value from rows and blend toward accent at 12%. Use fillRectBlend helper that reads existing row value and blends.
- 'dot_matrix': grid of 4x4 filled squares at 60px spacing. Same 12% blend.
- 'hex_grid': use hex center calculation (hexR=40, hexH=Math.sqrt(3)*hexR) — for each pixel, find nearest hex center, compute distance to hex boundary using Math.atan2 and borderDist check, mark as pattern line at 12% blend if borderDist >= 0 and borderDist < 2.
- 'topo': compute elevation = Math.sin(px*0.02 + 3*Math.sin(py*0.015)) + 0.7*Math.cos(px*0.03 + py*0.025). Mark pixel if Math.abs(elevation % 0.4) < 0.03. Same 12% blend.
All dark patterns write to bg_gradient.png. Light variants use light.bg base, light.accent at 8% blend.

Step 4 — Add generatePhotoBg(appDir, dark, light):
Check for shared/bg_photo.png. If absent: call generateGradientBg(appDir, dark, light); return. If present: read file with fs.readFileSync(), check PNG signature bytes [137, 80, 78, 71, 13, 10, 26, 10]. If signature mismatch: fall back to gradient. If signature OK: attempt minimal PNG decode — read IHDR chunk (width at bytes 16-19, height at bytes 20-23 as big-endian uint32). Read IDAT data, decompress with zlib.inflateSync(), reassemble scanlines assuming filter type 0 (None — no reconstruction needed). If any step throws: catch and fall back to generateGradientBg(appDir, dark, light). If decode succeeds: create 1920x1080 rgbRows using makeGradientRows as the base, then for each pixel compute bilinear sample from the decoded photo data, alpha-blend with overlay: photoR = samplePhoto(px, py, photoData, photoW, photoH); finalR = Math.round(photoR * 0.55 + baseR * 0.45). Write to bg_gradient.png. Light variant: use 0.15 photo weight (lighter overlay). Write to bg_gradient_light.png.

Step 5 — Add generateBackground(appDir, dark, light, visualLang) dispatcher:
  var bgType = (visualLang && visualLang.backgroundType) ? visualLang.backgroundType : 'gradient';
