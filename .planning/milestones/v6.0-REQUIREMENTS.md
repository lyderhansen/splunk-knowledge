# Requirements — Milestone v6.0 Speed & Oneshot

**Milestone goal:** splunk-custom-viz finishes a full session in fewer turns without hanging mid-code-write, AND a new oneshot-dashboard path produces a working Dashboard Studio dashboard from dummy data with zero design ceremony.

**Target:** `splunk-custom-viz` plugin (cv6, currently v6.0.7). `splunk-viz-packs` (v5.10.1) is legacy this milestone and out of scope.

---

## v6.0 Requirements

### CHUNK — Hang-free code writes (cv-create)

- [ ] **CHUNK-01**: User runs `cv-create` against any viz in the design lock and never observes a mid-file hang requiring restart — chunked / smaller-unit emission strategy applied to long viz files
- [ ] **CHUNK-02**: User running `cv-create` sees the skill emit each viz as a discrete unit (one viz = one self-contained write step), not as a single mega-output that can stall halfway
- [ ] **CHUNK-03**: User invoking `cv-create` after a previous interrupted run can pick up where it left off without re-emitting already-written vizs — per-viz isolation makes incremental progress durable

### PERF — End-to-end session reduction (splunk-custom-viz)

- [ ] **PERF-01**: User completes a full cv6 session (cv-scope → installable artifact, 3-5 vizs) in measurably fewer turns than the v6.0.7 baseline — verified by re-running an existing test prompt (test48 polestar or similar) and counting turns / total tokens
- [ ] **PERF-02**: User runs `cv-sketch` and the emitted `mockup.html` + `DESIGN-LOCK.md` contain only what's load-bearing for downstream `cv-create` — redundant / re-derivable content trimmed; design contract intact
- [ ] **PERF-03**: User runs each cv-* skill and the skill loads only the references it actually needs for the current step — no eager loading of recipes / blueprints / corrections that the step won't consume

### ONE — cv-oneshot skill (zero-ceremony dashboard)

- [ ] **ONE-01**: User runs `/cv-oneshot <brand or domain hint>` and receives an installable Dashboard Studio dashboard (JSON + minimal app wrapper) backed by inline `| makeresults | eval ...` SPL — no real Splunk data source required
- [ ] **ONE-02**: cv-oneshot produces output in the smallest possible number of user turns — ideally one turn after the initial prompt, no clarifying questions, no design ceremony, no validator repair loops
- [ ] **ONE-03**: User can swap the `makeresults` dummy SPL for real SPL post-generation by editing one clearly-marked location per panel — generated JSON keeps SPL delimited and editable, not hidden in deep template scaffolding
- [ ] **ONE-04**: cv-oneshot loads NONE of the design-principles / archetype / mood / aesthetic-flavor / DPR-* / D-* / score_design reference machinery — speed is the only success metric for this path

## Out of Scope (v6.0)

- Any v6 work on splunk-viz-packs — stays at v5.10.1 as legacy

- New domain-specific viz types — orthogonal to perf/hardening

- Validator / repair / autofix passes inside cv-oneshot — explicitly omitted to keep the path fast
