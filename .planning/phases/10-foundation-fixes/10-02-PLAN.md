    **ECR-06: hexFromSplunk on All Color Picker Reads**

    Requirement: Every formatter control using splunk-color-picker MUST have its opt()/getOption() value wrapped in hexFromSplunk(). Splunk delivers color picker values as decimal integers (e.g., 30646) not hex strings. Direct use of the raw value as ctx.fillStyle renders black or wrong color.

    Wrong pattern:
    var zoneColor = opt('detractorColor', '#FF4136');
    ctx.fillStyle = zoneColor;  // BUG: zoneColor may be "30646", not "#FF4136"

    Correct pattern:
    var zoneColor = hexFromSplunk(opt('detractorColor', ''), t.error);
    ctx.fillStyle = zoneColor;  // CORRECT: hexFromSplunk converts integer to hex

    Rules:
    - The fallback in hexFromSplunk should be a theme token (t.error, t.accent, etc.), not a hardcoded hex string, so the fallback adapts to the current theme.
    - The opt() fallback for color pickers should be empty string (''), letting hexFromSplunk handle the default via its own fallback parameter.
    - This applies to ALL color pickers: accentColor, zone colors, series colors, status colors.
    - Root cause: broken-rules.md B22.

    **ECR-07: showHoverEffect Early-Exit in _onMouseMove**

    Requirement: When showHoverEffect=false, the _onMouseMove handler MUST return immediately without calling invalidateUpdateView(). Without this, mouse movement triggers 60fps redraws even though no hover highlight is rendered.

    Wrong pattern:
    _onMouseMove: function(e) {
        var newIndex = this._getHitIndex(e);
        if (newIndex !== this._hoveredIndex) {
            this._hoveredIndex = newIndex;
            this._hoverTarget = (newIndex >= 0) ? 0.12 : 0;
            this._startHoverTransition(this._lastConfig, ns);
        }
    }
    // BUG: _startHoverTransition checks showHoverEffect but _onMouseMove already
    // triggered invalidateUpdateView in some implementations

    Correct pattern:
    In updateView, store the hover flag:
    this._showHoverEffect = opt('showHoverEffect', 'true') === 'true';

    In _onMouseMove, early-exit:
    _onMouseMove: function(e) {
        if (!this._showHoverEffect) return;
        // ... hit test, update hover state, invalidate ...
    }

    Rules:
    - Store _showHoverEffect as an instance property in updateView (not re-read from config in _onMouseMove, because _onMouseMove does not receive config).
    - The early return prevents ALL downstream processing: no hit testing, no hover state changes, no invalidateUpdateView.
    - This is a performance fix: on a 10-viz dashboard, unnecessary redraws from mouse movement cause visible frame drops.

    All code examples must be pure ES5.
