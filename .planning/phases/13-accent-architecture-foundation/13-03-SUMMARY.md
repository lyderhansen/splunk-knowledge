# Phase 13 Plan 03: Silhouette Expansion and Contrast Guard Summary

Expanded `generate_assets.js` VIZ_TYPE_KEYWORDS from 8 to 10 types with domain-specific compound keyword coverage, added `drawScatterSilhouette` and `drawNetworkSilhouette` drawing functions, and added `previewContrastAccent` contrast guard so low-contrast accent colors are brightened 30% toward white before silhouette rendering.

## What Was Built

**Task 1 — VIZ_TYPE_KEYWORDS expansion (8 → 10 types):**

- `scatter` type added: `latency_scatter`, `scatterplot`, `bivariate`, `distribution`, `scatter`, `bubble`, `correlation`, `plot`, `xy`
- `network` type added: `network_topology`, `force_directed`, `dependency`, `relationship`, `geographic`, `flight`, `route`, `network`, `topology`, `connection`, `graph`, `force`, `node`, `edge`, `path`
- Existing types expanded with compound domain-specific keywords: `passenger_flow` → timeline, `attack_heatmap` → grid, `leaderboard` → bars, `kpi_gauge` → gauge, `power_horizon` → line, `stage_tracker` → progress
- Bare `map` keyword removed from grid type — prevented `network_topology` from falsely matching grid
- Compound keywords ordered before their short-prefix components within each type array
- `kpi` remains last entry as default fallback

**Task 2 — New silhouette functions and contrast guard:**

- `drawScatterSilhouette`: draws x/y axis lines and 12 data point dots of varying sizes (r=5..14)
- `drawNetworkSilhouette`: draws hub-to-spoke edges (Bresenham-style line stepping), a peripheral ring connecting outer nodes, and filled node squares (hub=8px, spokes=5px)
- `drawSilhouette` dispatcher updated with `scatter` and `network` branches before the `kpi` fallback
- `previewContrastAccent(accentHex, bgHex)`: computes perceived luminance ratio; if below 3:1, returns accent brightened 30% toward white; otherwise returns original accent RGB
- `generatePreviews` now calls `previewContrastAccent(dark.accent, dark.bg)` instead of `hexToRgb(dark.accent)` directly
