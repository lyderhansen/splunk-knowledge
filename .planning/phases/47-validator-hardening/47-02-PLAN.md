<objective>
Build a dev-local sweep helper (tests/validate_sweep.sh) that runs the Phase 47-hardened validate.sh against every in-repo viz pack, then execute the sweep and verify zero unexpected FAILs. This is the regression gate that closes VAL-05 — the contract that K1b/K5/K6/K7 do not silently break packs that were passing K1/K2/K3 cleanly before.

Purpose: Plan 47-01 added four new checks based on test51 + test52 evidence. Before declaring v6.0.9 ready to ship, we need empirical proof those checks behave correctly across the full ~42 modern test pack inventory (researcher A2 verified count). Expected NEW FAILs (the actual bugs the checks were built to catch) are fine; UNEXPECTED FAILs on previously-clean packs would mean the heuristic is too aggressive — gate that finding before merge.

Output: tests/validate_sweep.sh (gitignored, dev-only — tests/ is in .gitignore per CONTEXT.md / repo memory; the script is a local convenience, NOT bundled in the plugin), sweep-output.txt for the SUMMARY.md report, and an updated 47-02-SUMMARY.md with the per-pack pass/FAIL breakdown.
</objective>

    2. Discovers candidate viz packs by finding every directory containing `appserver/static/visualizations/` under `tests/` and `.cv/` (if present). Use `find` with `-type d -name visualizations -path '*/appserver/static/*'` then sed-strip the suffix to get the pack root.

    8. Exit code: 0 if zero UNEXPECTED FAILs (all NEW-code FAILs are on test51 or test52); 1 if any pack has a NEW-code FAIL outside those two source-failure-mode packs (means the heuristic is over-firing).

    - macOS BSD-compatible (no `-P`, no GNU-only `find` flags). `mapfile` is bash 4+ — macOS ships bash 3.2; use `while IFS= read -r line; do ARR+=("$line"); done < <(find ...)` instead, OR use `xargs`.
